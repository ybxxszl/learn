Web服务 - Apache
	1、Web服务原理、HTTP协议
		Web服务端与客户端之间通过HTTP协议传输数据
			当前使用最多的HTTP版本是HTTP/1.1，HTTP/2.0正在开发
			HTTP使用TCP协议，默认端口号为80
			HTTP方法
				GET	从服务器上获取一个资源（一般是网页）
				POST	向服务器提交数据（可能会创建一个新的资源）
				PUT	向服务器提交数据
				DELETE	删除指定资源
				HEAD	只请求网页的头部信息
			HTTP状态码
				200	正常，请求成功
				301	永久移动，一般用于域名重定向
				304	未修改，一般用于缓存
				401	禁止访问，未授权
				403	禁止访问，通常代表已认证通过，但是没有访问权限
				404	未找到资源
				500	服务器内部错误
		Web服务架构
			Web浏览器  HTTP  Web Server AppServer DB Server
	2、Apache服务基础配置
		特点：
			动态的、预创建的进程 动态模块加载 虚拟主机 SSL主机
		命令：
			可以通过以下命令安装apache软件：
				yum install -y httpd
			安装之后，通过以下命令启动apache:
				service httpd start
			apache提供的是Web服务，所以默认使用80(http)和443(https)端口（加密为443）
			apache的配置文件保存在：
				/etc/httpd/conf/httpd.conf
			apache模块的配置文件保存在：
				/etc/httpd/conf.d/
			默认网站日志为：
				/var/log/httpd/
			网页服务：默认apache不需要配置即可启动，默认网站的根目录为/var/www/html
		配置：
			KeepAlive off						tcp持续连接功能
			MaxKeepAliveRequest 100					连接的最大请求数
			KeepAliveTimeout 15					持续连接等待时间
			Listen 80						提供服务的端口号（可以为192.168.56.1:80，只对这个地址的端口提供服务）
			LoadModule auth_basic_module modules/mod_auth_basic.so	加载模块
			User apache						运行程序的用户身份
			Group apache						运行程序的用户组
			ServerAdmin root@linuxcast.net				管理员邮箱（当网站出现错误，返回管理员邮箱，提示联系管理员）
			DocumentRoot "/var/www/html"				默认网站根目录路径，如果需要修改，则全部修改
			<Directory "/var/www/html"></Directory>			默认网站根目录路径，如果需要修改，则全部修改
			DirectoryIndex index.html index.html.var		不输入网页名称，默认显示的网页，空格隔开，从左到右
		语法检查：
			service httpd configtest
			apachectl configtest
			httpd -t
	3、Apache服务配置 - HTTP认证
		可以配置apache为特定文件夹进行简单的HTTP用户认证，已达到简单的访问控制权限
			1、创建apache账号、密码文件：
				htpasswd -cm /etc/httpd/.htpasswd linuxcast
				htpasswd -m /etc/httpd/.htpasswd nash_su
			2、修改apache配置文件，为特定目录配置认证：
				<Directory /var/www/linuxcast>
					AuthName "Linuxcast Auth"
					AuthType basic
					AuthUserFile /etc/httpd/.htpasswd
					Require valid-user
				</Directory>
	4、Apache服务配置 - 虚拟主机
		默认apache的配置只提供了一个网站服务，但是我们可以配置一台apache服务器为多个网站提供服务，我们称之为虚拟主机
		虚拟主机分为两种：
			基于IP的虚拟主机
				apache服务通过不同的访问IP区分不同的虚拟主机
			基于域名的虚拟主机
				apache服务通过不同的访问域名区分不同的虚拟主机
		一般使用基于域名的虚拟主机，但是如果需要搭建SSL的虚拟主机，则必须使用基于IP的虚拟主机
		基于域名的虚拟主机
			确保DNS服务器将不同的域名解析到同一个IP地址上
			使用命令host或dig确定解析正常
				host www1.linuxcast.net
				host www2.linuxcast.net
			在apache配置文件中添加如下配置
				NameVirtualHost *:80
				<VirtualHost *:80>
					ServerName www1.linuxcast.net
					ServerAdmin root@linuxcast.net
					DocumentRoot /var/www/linuxcast-1
				</VirtualHost>
				<VirtualHost *:80>
					ServerName www2.linuxcast.net
					ServerAdmin root@linuxcast.net
					DocumentRoot /var/www/linuxcast-2
				</VirtualHost>
			确保apache用户对网页根目录具有访问权限
Apache专题
	Apache服务的安装、管理及相关文件
		yum list | grep httpd 查找httpd软件包
		service httpd graceful 完成所有请求后重启服务器
		service httpd graceful 完成所有请求后关闭服务器
		相关文件：
			/etc/httpd/conf/目录下		主配置文件
			/etc/httpd/conf.d/目录下	模块配置文件
			/etc/httpd/logs/目录下		日志文件
			/etc/httpd/modules/目录下	模块文件
			/etc/httpd/run/目录下		运行信息
			/var/www/目录下			网站根目录
			/var/www/html/目录下		默认网页位置
	Apache服务基本概念（DSO、MPM等）
		apache默认监听TCP协议80端口
		apache服务默认会启动一个主进程（控制进程）和多个子进程，以root身份运行的失主进程，以apache身份运行的是紫禁城。主进程的进程ID保存在/etc/httpd/run/httpd.pid文件内
		查看apache相关进程
			ps aux | grep httpd
		查看80端口
			netstat -tupln | grep 80
		安装httpd的全部软件包
			yum install -y httpd-*
		Apache DSO
			apache是一个模块化设计的服务，核心只包含主要功能，扩展功能通过模块实现。不同的模块可以被静态的编译进程序，也可以动态加载，通过DSO实现
			查看所有模块
				httpd -M
			查看静态编译进程序的模块
				httpd -l
			编译模块的命令
				apxs
		Apache MPM
			MPM负责实现网络监听、请求的处理等功能，MPM有很多种，为了在不同平台环境下实现最优化的性能和最强的稳定性
			查看当前apache服务所使用的MPM
				httpd -i
				httpd -V
			prefork(子进程指数级增长)：
				<IfModule prefork.c>
				StartServers       8		默认子进程数
				MinSpareServers    5		最小空闲进程数
				MaxSpareServers   20		最大空闲进程数
				ServerLimit      256		并发数限制
				MaxClients       256		最大并发数，需要小于等于并发数限制
				MaxRequestsPerChild  4000	每个子进程最多处理请求数
				</IfModule>
			top命令动态查看系统运行状态

CentOS6.9安装MySQL
	1、检查MySQL是否已经安装
		yum list installed mysql*
	2、卸载已经安装的mysql（建议使用yum命令，因为yum命令可以自动删除与mysql相关的依赖；如果使用rpm命令，则还需要手动删除和mysql相关的文件）
		yum remove mysql-libs.x86_64
	3、查找mysql相关文件
		whereis mysql
	4、查看系统中的mysql安装包
		yum list mysql*
	5、安装mysql-server
		yum install -y mysql-server
	6、设置3306端口，并保存
		/sbin/iptables -I INPUT -p tcp --dport 3306 -j ACCEPT
		/etc/rc.d/init.d/iptables save
	7、设置mysql开机启动
		chkconfig mysqld on
	8、启动mysql服务
		service mysqld start
	9、连接mysql
		mysql
	10、为mysql设置密码
		use mysql; --使用数据库
		update user set password=password('root') where user='root';
		flush privileges; --刷新权限
	11、退出mysql
		exit
	12、登录mysql数据库
		mysql -hlocalhost -uroot -proot -P3306 或 mysql -uroot -proot
